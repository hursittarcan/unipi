Logical Errors in Python

Understanding and fixing logical errors in Python selection structures

What are Logical Errors?

Logical errors occur when your code runs without syntax errors but produces incorrect results due to flawed logic.

01

No error messages

02

Code executes completely

03

Incorrect output

Common Logical Errors in Python

1. Incorrect Comparison Operators

Incorrect

# Check if user is an adult
age = 25
if age = 18:
    print("You are an adult")

SyntaxError: invalid syntax

In Python, using = (assignment) instead of == (comparison) causes a syntax error, unlike some other languages where it would be a logical error.

Correct

# Check if user is an adult
age = 25
if age >= 18:
    print("You are an adult")

Use == for equality comparison and >= for "greater than or equal to".

2. Incorrect Indentation Logic

Incorrect

# Check weather conditions
is_raining = True
is_cold = True

if is_raining:
    print("Bring an umbrella")
    if is_cold:
        print("Bring a jacket")
print("Wear boots")  # Always executes

The "Wear boots" instruction will always execute regardless of weather conditions due to incorrect indentation.

Correct

# Check weather conditions
is_raining = True
is_cold = True

if is_raining:
    print("Bring an umbrella")
    if is_cold:
        print("Bring a jacket")
    print("Wear boots")  # Only if raining

Proper indentation ensures "Wear boots" only executes when it's raining.

3. Overlapping Conditions

Incorrect

# Grade calculator
score = 85

if score >= 70:
    grade = "C"
if score >= 80:
    grade = "B"
if score >= 90:
    grade = "A"

print(f"Your grade is {grade}")

For score = 85, all conditions are checked independently, resulting in grade = "B" (not "C" as expected).

Correct

# Grade calculator
score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"

print(f"Your grade is {grade}")

Using elif ensures mutually exclusive conditions, giving the correct grade.

Fix the Logical Error

This function should determine if a number is prime, but it has a logical error. Can you fix it?

Hint: You don't need to check all numbers from 2 to num-1. Think about the largest possible factor of a number.

Solution:

def is_prime(num):
    """Check if a number is prime."""
    # A prime number is greater than 1 and only
    # divisible by 1 and itself

    if num <= 1:
        return False

    # We only need to check up to the square root of num
    for i in range(2, int(num**0.5) + 1):
        if num % i == 0:
            return False

    return True

The optimization: We only need to check divisibility up to the square root of the number, not all the way to num-1.
