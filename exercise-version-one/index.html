Common Logical Errors

Incorrect Comparison Operators

Using the wrong comparison operator can lead to unexpected results.

// Intended: Check if age is exactly 18
if (age = 18) { // WRONG! This assigns 18 to age
    console.log("You're 18!");
}
// Correct version
if (age === 18) { // Correct comparison
    console.log("You're 18!");
}

Boundary Conditions

Errors often occur at the boundaries of conditions.

// Intended: Check if score is passing (60 or above)
if (score > 60) { // WRONG! Excludes exactly 60
    console.log("You passed!");
}
// Correct version
if (score >= 60) { // Includes 60
    console.log("You passed!");
}

Overlapping Conditions

Multiple conditions that overlap can cause unexpected behavior.

// Grading system
if (score >= 90) {
    grade = "A";
}
if (score >= 80) { // WRONG! This will always execute if score >= 90
    grade = "B";
}
// Correct version
if (score >= 90) {
    grade = "A";
} else if (score >= 80) {
    grade = "B";
}
