Logical Errors in Selection Structures

A learning resource for programming students

What are Logical Errors?

Logical errors occur when your code syntax is correct (it runs without crashing), but the program's behavior is incorrect due to flaws in your logic. These errors are often harder to spot because the program still executes without error messages.

Common Logical Errors in Selection Structures:

  • Using assignment (=) instead of comparison (==, ===)
  • Incorrect compound conditions with AND (&&) and OR (||)
  • Improper nesting of if/else statements
  • Missing else clauses creating logic gaps

Common Error Examples

1. Assignment vs. Comparison

Incorrect Code:

// Check if age is 18
if (age = 18) {
    console.log("You are 18 years old");
}

This code assigns 18 to age and always evaluates to true

Correct Code:

// Check if age is 18
if (age === 18) {
    console.log("You are 18 years old");
}

This correctly compares age to 18

2. Independent vs. Mutually Exclusive Conditions

Incorrect Code:

// Determine grade
if (score < 60) {
    grade = "F";
}
if (score < 70) {
    grade = "D";
}
if (score < 80) {
    grade = "C";
}

For score = 65, grade will be "D", but for score = 55, it will run through all conditions and grade will end up as "C"

Correct Code:

// Determine grade
if (score < 60) {
    grade = "F";
} else if (score < 70) {
    grade = "D";
} else if (score < 80) {
    grade = "C";
}

Using else-if ensures each score only matches one grade range

3. Missing Braces and Improper Nesting

Incorrect Code:

// Apply discount
if (isMember)
    if (isPremium)
        discount = 20;
else
    discount = 5;

The else attaches to the nearest if (isPremium), so non-premium members get 5% discount but non-members get no discount

Correct Code:

// Apply discount
if (isMember) {
    if (isPremium) {
        discount = 20;
    } else {
        discount = 10;
    }
} else {
    discount = 5;
}

Proper braces clarify the structure: premium members get 20%, regular members 10%, and non-members 5%
