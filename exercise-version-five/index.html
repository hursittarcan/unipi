What are Logical Errors? ü§î

Logical errors are bugs in my code that don't cause syntax errors or crashes, but make my program behave incorrectly. The code runs, but it doesn't do what I want it to do!

Remember: Logical errors are the hardest to find because the program doesn't tell me there's a problem!

Common Logical Errors in If Statements ‚ö†Ô∏è

  • Using = (assignment) instead of == or === (comparison)
  • Forgetting to use curly braces { } with multi-line blocks
  • Using separate if statements when I need else-if
  • Incorrect boolean logic with AND (&&) and OR (||)
  • Off-by-one errors in boundary conditions

Error #1: Assignment vs. Comparison üîÑ

// This is WRONG!

if (x = 10) {
    console.log("x is 10");
}

This assigns 10 to x and always evaluates to true!

‚Üí

// This is RIGHT!

if (x === 10) {
    console.log("x is 10");
}

This checks if x equals 10

= is assignment

== is comparison (value)

=== is strict comparison (value AND type)

Error #2: Missing Braces üìé

// This is WRONG!

if (isRaining)
    console.log("Take umbrella");
    console.log("Wear boots");

Only the first line is in the if block!

‚Üí

// This is RIGHT!

if (isRaining) {
    console.log("Take umbrella");
    console.log("Wear boots");
}

Both lines are in the if block

Always use { } for clarity!

Error #3: Cascading If vs. Else-If üåä

// This is WRONG!

// Grade calculator
if (score >= 90) {
    grade = "A";
}
if (score >= 80) {
    grade = "B";
}
if (score >= 70) {
    grade = "C";
}

A score of 95 will get a C grade!

‚Üí

// This is RIGHT!

// Grade calculator
if (score >= 90) {
    grade = "A";
} else if (score >= 80) {
    grade = "B";
} else if (score >= 70) {
    grade = "C";
}

Now a score of 95 gets an A grade

Use else-if for mutually exclusive conditions!

Try It: Fix the Logical Error ‚úèÔ∏è

This function should tell if a year is a leap year. Can you fix the logical error?

Think about the order of conditions. If a year is divisible by both 4 and 100, which condition runs first?

Also, if a year is divisible by 400, it's also divisible by 100 and 4. Will that code ever run?

function isLeapYear(year) {
    // A leap year is divisible by 4
    // But century years must also be divisible by 400

    if (year % 400 == 0)
        return true;
    if (year % 100 == 0)
        return false;
    if (year % 4 == 0)
        return true;

    return false;
}

The order matters! Check the most specific condition first (divisible by 400).

My Notes üìí

To avoid logical errors in selection structures:

  1. Always double-check my comparison operators (==, ===, not =)
  2. Use curly braces {} even for single-line if statements
  3. Think about the order of conditions and use else-if for mutually exclusive options
  4. Test boundary conditions (like 0, 100, etc.)
  5. Draw out the logic flow if I'm confused
